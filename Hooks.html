Hooks and Lifecycle Methods

Hooks

What are hooks?

Hooks allow function components to access properties and are the reason class components are less necessary. In a class component, you would have a constructor that contains your states and set methods to change them. This functionality is simplified in components using hooks.

In this class component example, Car is created with a contructor that defines the state and includes the method setColor which changes the color state when called. 

            
                class Car extends React.Component{
                    constructor(props) {
                        super(props);
                        this.state = {
                            brand: "Ford",
                            model: "Mustang",
                            color: "red",
                            year: 1964
                            };
                        }
                    setColor = () => {
                        this.setState({color: "blue"});
                    }
                render(){
                    <>
                        <h1>My car is {color}!</h1>
                        <button
                            type="button"
                            onClick={this.setColor};
                        >Blue</button>
                    </>
                }
            }

useState

The useState hook is used to track states in a fucntion component. It accecpts an intital state and returns an array of two values, the current state and a a function that updates the state, like the class component's set state method. Hooks use destructuring assignment to unpack the array into seperate varibles In the example below, we create a use state hook in the function component Car.

The first value color is set to the current state of red. The second value setColor it the function used to change the state. 

            
import { useState } from "react";
    function Car() {
    const [color, setColor] = useState("red");

)}
